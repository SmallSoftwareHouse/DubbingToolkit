# 架构与技术参考

本文档描述项目的内部结构、主要模块、开发规范及组件状态，主要面向参与开发或希望了解系统内部运行机制的人员。

## 文件夹结构

```
DubbingToolkit/
├── Billing/                TTS 消耗和成本监控
├── core/                   共享 Python 辅助模块
├── credentials/            API 凭据（已从 Git 排除）
├── Installation/           本地 Python 运行时（3.10、3.11）
├── installer/              构建和打包系统
├── locale/                 本地化和语言管理
│   ├── Active/             活跃语言 JSON 文件（it、en、es、de、fr、pt、ru、zh）
│   └── System/             语言元数据（Whisper、支持的语言）
├── Logs/                   运行日志
├── ps/                     PowerShell 模块（日志、消息）
├── Repository/             共享资源和本地模型
├── Scripts/                运行脚本和 Python 模块
│   └── maintenance/        维护脚本和本地化管道
├── Settings/               活跃配置和参考配置
├── Temp/                   临时文件
├── Tools/                  外部二进制文件（ffmpeg）
├── venv/                   主 Python 虚拟环境
├── voices/                 可用 TTS 语音和音频示例
└── Workspace/              项目数据（自动创建）
    └── projects/
        └── {项目名称}/
            ├── project_info.json                    项目元数据
            ├── audio_extraction/
            │   ├── current/                         当前音频轨道
            │   └── archive/                         提取历史
            ├── transcripts/
            │   ├── current/                         当前 SRT 转录
            │   └── archive/                         转录历史
            ├── translated/
            │   ├── current/                         当前 SRT 翻译
            │   └── archive/                         翻译历史
            ├── dubbed/
            │   ├── current/                         当前 TTS 音频
            │   └── archive/                         TTS 音频历史
            ├── video_input/                         源视频（从不修改）
            └── audio_input/                         直接音频输入（可选）
```

## 启动链

```
StartDubbing.bat
  └→ Scripts/Launcher.ps1
       激活 venv、UTF-8 设置、日志、语言加载
         └→ Scripts/Regista.py
              主菜单和管道编排
```

Launcher 管理：本地 Python 运行时选择（`Installation/`）、venv 创建/激活、日志系统设置、界面语言加载。

`Regista.py` 是核心协调器：向用户呈现菜单，并将执行委托给每个阶段的特定模块。

## 运行管道

| 阶段 | 模块 | 输入 → 输出 |
|---|---|---|
| 1 — 音频提取 | `Scripts/estrai_tracce.py` | `video_input/` → `audio_extraction/current/` |
| 2 — 转录 | `Scripts/trascrivi_audio.py` | `audio_extraction/current/` 或 `audio_input/` → `transcripts/current/`（SRT） |
| 3 — 翻译 | `Scripts/traduci_testo.py` | `transcripts/current/` → `translated/current/`（SRT） |
| 4 — TTS | `Scripts/tts_menu.py` | `translated/current/` → `dubbed/current/`（MP3/WAV） |

所有路径均相对于 `Workspace/projects/{项目名称}/`。`tts_menu.py` 根据活跃提供商委托给 `tts_azure.py` 或 `tts_google.py`。

## Core 模块（`core/`）

| 模块 | 功能 |
|---|---|
| `messages.py` | 集中本地化消息 — 读取 `locale/Active/<lang>.json` |
| `credentials_manager.py` | API 凭据的加载和验证 |
| `api_check.py` | 访问 TTS 菜单前的凭据验证 |
| `logger.py` | 结构化会话日志（INFO/WARN/ERROR），JSON 格式 |
| `error_reporter.py` | 错误报告系统：日志 ZIP、mailto: 开发者 |
| `update_checker.py` | 在 GitHub Releases 上检查更新 |
| `ui_printer.py` + `ui_colors.py` | 控制台格式化和颜色 |
| `utils_tts.py` | SRT 解析共享工具 |
| `workspace_manager.py` | 活跃 workspace 管理、stage 结构、档案轮换 |
| `source_importer.py` | 将外部文件导入 workspace 的对话框 |

## 本地化

```
locale/
├── Active/              活跃语言文件（运行时）
│   ├── it.json, en.json, es.json, de.json, fr.json, pt.json, ru.json, zh.json
└── System/
    ├── languages.json           概念上支持的语言
    └── whisper_languages.json   Whisper 支持的语言
```

- 所有 Python 界面消息使用 `core/messages.py`。
- `locale/Active/` 中的所有文件必须同步。
- 缺失的键在运行时产生 `[MISSING: key]`。
- PowerShell 使用 `ps/Messages.psm1`。
