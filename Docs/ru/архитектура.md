# Архитектура и техническое описание

Этот документ описывает внутреннюю структуру проекта, основные модули, соглашения о разработке и состояние компонентов. Предназначен прежде всего для тех, кто участвует в разработке или хочет понять внутреннее устройство системы.

---

## Структура папок

```
DubbingToolkit/
├── Billing/                Мониторинг потребления и затрат TTS
├── core/                   Общие вспомогательные модули Python
├── credentials/            Учётные данные API (исключены из Git)
├── Installation/           Локальные среды выполнения Python (3.10, 3.11)
├── installer/              Система сборки и упаковки
├── locale/                 Локализация и управление языками
│   ├── Active/             Активные JSON-файлы языков (it, en, es, de, fr, pt, ru, zh)
│   └── System/             Метаданные языков (Whisper, поддерживаемые языки)
├── Logs/                   Рабочие журналы
├── ps/                     Модули PowerShell (журналирование, сообщения)
├── Repository/             Общие ресурсы и локальные модели
├── Scripts/                Рабочие скрипты и модули Python
│   └── maintenance/        Скрипты обслуживания и pipeline локализации
├── Settings/               Активная и справочная конфигурация
├── Temp/                   Временные файлы
├── Tools/                  Внешние бинарные файлы (ffmpeg)
├── venv/                   Основная виртуальная среда Python
├── voices/                 Доступные голоса TTS и аудиообразцы
└── Workspace/              Данные проектов (создаётся автоматически)
    └── projects/
        └── {имя_проекта}/
            ├── project_info.json                    Метаданные проекта
            ├── audio_extraction/
            │   ├── current/                         Текущие аудиодорожки
            │   └── archive/                         Архив предыдущих извлечений
            ├── transcripts/
            │   ├── current/                         Текущие транскрипции SRT
            │   └── archive/                         Архив транскрипций
            ├── translated/
            │   ├── current/                         Текущие переводы SRT
            │   └── archive/                         Архив переводов
            ├── dubbed/
            │   ├── current/                         Текущий аудиофайл TTS
            │   └── archive/                         Архив TTS
            ├── video_input/                         Исходное видео (никогда не изменяется)
            └── audio_input/                         Прямой аудиовход (необязательно)
```

## Цепочка запуска

```
StartDubbing.bat
  └→ Scripts/Launcher.ps1
       Активирует venv, настройка UTF-8, журналы, загрузка языка
         └→ Scripts/Regista.py
              Главное меню и оркестрация pipeline
```

Launcher управляет: выбором локальной среды выполнения Python (`Installation/`), созданием/активацией venv, настройкой системы журналов, загрузкой языка интерфейса.

`Regista.py` — центральный координатор: отображает меню пользователю и делегирует выполнение конкретным модулям для каждого этапа.

## Рабочий pipeline

| Этап | Модуль | Вход → Выход |
|---|---|---|
| 1 — Извлечение аудио | `Scripts/estrai_tracce.py` | `video_input/` → `audio_extraction/current/` |
| 2 — Транскрипция | `Scripts/trascrivi_audio.py` | `audio_extraction/current/` или `audio_input/` → `transcripts/current/` (SRT) |
| 3 — Перевод | `Scripts/traduci_testo.py` | `transcripts/current/` → `translated/current/` (SRT) |
| 4 — TTS | `Scripts/tts_menu.py` | `translated/current/` → `dubbed/current/` (MP3/WAV) |

Все пути относительны к `Workspace/projects/{имя_проекта}/`. `tts_menu.py` делегирует `tts_azure.py` или `tts_google.py` в зависимости от активного провайдера.

## Модули core (`core/`)

| Модуль | Функция |
|---|---|
| `messages.py` | Централизованные локализованные сообщения — читает `locale/Active/<lang>.json` |
| `credentials_manager.py` | Загрузка и валидация учётных данных API |
| `api_check.py` | Проверка учётных данных перед доступом к меню TTS |
| `logger.py` | Структурированный журнал сеанса (INFO/WARN/ERROR) в формате JSON |
| `error_reporter.py` | Система отчётов об ошибках: ZIP журналов, mailto: разработчику |
| `update_checker.py` | Проверка обновлений на GitHub Releases |
| `ui_printer.py` + `ui_colors.py` | Форматирование и цвета консоли |
| `utils_tts.py` | Общие утилиты для парсинга SRT |
| `workspace_manager.py` | Управление активным workspace, структура stages, ротация архивов |
| `source_importer.py` | Диалог импорта внешних файлов в workspace |

## Локализация

```
locale/
├── Active/              Активные языковые файлы (runtime)
│   ├── it.json, en.json, es.json, de.json, fr.json, pt.json, ru.json, zh.json
└── System/
    ├── languages.json           Концептуально поддерживаемые языки
    └── whisper_languages.json   Языки, поддерживаемые Whisper
```

- Все сообщения интерфейса Python используют `core/messages.py`.
- Все файлы в `locale/Active/` должны быть синхронизированы.
- Отсутствующие ключи производят `[MISSING: key]` во время выполнения.
- PowerShell использует `ps/Messages.psm1`.
